# Foundry Doctor Reports

This directory contains root cause analysis reports generated by the u-doctor agent.

## Structure

- `root-cause-{YYYYMMDD-HHMMSS}.md` - Individual root cause reports
- `patterns.md` - Auto-generated summary of recurring issues

## Usage

Run doctor diagnostics:
```bash
/doctor              # Diagnose and create report
/doctor analyze      # Analyze patterns
/doctor fix          # Apply state fixes
```

Or run directly:
```bash
opencode run --agent u-doctor "Diagnose current Foundry state"
```

## Report Format

Each root-cause report contains:
- **Symptoms**: Observable problems
- **Affected Tasks**: Which tasks are impacted
- **Root Cause**: Why it happened
- **Evidence**: State files, logs, process info
- **Pattern Analysis**: Similar issues from history
- **Recommended Fix**: State fixes (immediate) and code fixes (long-term)

## Workflow

1. **Diagnosis**: Doctor identifies issues
2. **Documentation**: Creates root-cause-{timestamp}.md
3. **Pattern Detection**: Checks for recurring issues
4. **State Fix**: Applies immediate fixes (locks, zombies, state.json)
5. **Code Fix**: When 3+ similar reports → recommend code change

## Common Issues

See individual root-cause reports for details on:
- Stale locks
- Zombie processes
- Missing files (handoff.md, summary.md)
- Model stalls
- Branch deletion issues
- Preflight failures
