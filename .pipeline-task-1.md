# Fix: E2E agent health badge not showing after discovery

Агенти видимі на сторінці `/admin/agents`, але не мають `badge-healthy` або `badge-degraded`.
Тести `hello_agent_test.js` та `news_maker_admin_test.js` фейляться на перевірці health badge.

## Проблема
Після discovery агенти з'являються в таблиці, але їх health badge не відображається.
XPath `//tr[contains(@data-agent-name,"hello-agent")]//span[contains(@class,"badge-healthy")]` не знаходить елемент.

## Можливі причини
1. Discovery через Traefik API не знаходить агентів (E2E контейнери не мають Traefik labels)
2. Health check в core використовує `localhost:PORT` замість Docker DNS для E2E контейнерів
3. Badge рендериться тільки після successful health poll, який може не встигнути

## Вимоги
- Перевірити що core-e2e може виконати health check до кожного агент-контейнера
- Переконатись що discovery в E2E графі працює (agent URLs зареєстровані правильно)
- Якщо потрібно — збільшити timeout в тестах для health poll
- Тести повинні бачити `badge-healthy` або `badge-degraded` для зареєстрованих агентів

## Файли для аналізу

- E2E агенти зареєстровані через `/api/v1/internal/agents/register` з правильними URLs
- Health endpoints всіх агентів відповідають OK (перевірено вручну через curl)
