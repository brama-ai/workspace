# Fix: E2E tests docker compose exec fails inside devcontainer

E2E тести використовують `docker compose exec -T postgres psql` для seed/cleanup даних,
але всередині devcontainer ця команда не знаходить compose файлів (`no configuration file provided: not found`).

## Проблема
Тести в `tests/e2e/tests/admin/chats_test.js`, `dashboard_test.js`, `coder_events_test.js`,
`log_trace_test.js` та `openclaw/core_db_isolation_test.js` викликають `docker compose exec` з CWD = `tests/e2e/`,
але compose файли знаходяться в `docker/` у корені проекту.

## Вимоги
- Замінити `docker compose exec -T postgres psql` на `docker exec brama-postgres-1 psql` у всіх тест-файлах
- Або передавати правильний `-f` шлях до compose файлів
- Перевірити що `docker compose --profile e2e exec -T` також замінено

## Файли для зміни
- `core/tests/e2e/tests/admin/chats_test.js`
- `core/tests/e2e/tests/admin/dashboard_test.js`
- `core/tests/e2e/tests/admin/coder_events_test.js`
- `core/tests/e2e/tests/admin/log_trace_test.js`
- `core/tests/e2e/tests/openclaw/core_db_isolation_test.js`
- `core/tests/e2e/tests/smoke/agent_chat_test.js`

## Контекст
- 16+ тестів зафейлились з `no configuration file provided: not found`
- Compose файли перемістились з кореня в `docker/` директорію
- `docker exec <container-name>` працює незалежно від CWD
